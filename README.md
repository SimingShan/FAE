# WFAE — FAE: a Function AutoEncoder for PDE Representation Learning

Research codebase for **FAE (Function AutoEncoder)** — a sparse, function-space
representation learner for PDE fields — and the **G1 benchmark**, an evaluation
framework built on the position that reconstruction accuracy is a weak proxy
for representation quality.

## The two core methods

| | |
|---|---|
| **FAE + VICReg** (deterministic) | Perceiver-style encoder over coordinate–value tokens (arbitrary sensor count N), Senseiver-style cross-attention decoder. Trained with two-view multi-count reconstruction + VICReg variance/covariance through a projector. `src/models/fae.py`, `scripts/train_fae.py --method fae_vicreg` |
| **FAE-NP** (probabilistic) | Same encoder, pooled to a **single global Gaussian latent** z; heteroscedastic cross-attention decoder; NP ELBO with KL(q(z\|C,T)‖q(z\|C)) + free bits. `src/models/fae_np.py`, `scripts/train_fae_np.py` |

Baselines at ~7M parameter parity: `MLPSparseAE` (set-pool), `CNN1DAE`,
`MAE1DAE`, JEPA-Perceiver, JEPA-ViT (`src/models/baselines.py`,
`src/models/jepa_vit.py`).

## The G1 benchmark (1D multi-PDE)

Four systems, 5000 trajectories × 100 frames × 1024 points each, continuous
coefficient sampling, generated by `data_gen/gen_g1_all.py`:

| system | equation | coefficient |
|---|---|---|
| heat | u_t = ν u_xx | ν ~ logU[1e-3, 1e-1] |
| advection | u_t + β u_x = 0 | β ~ U[0.1, 4] |
| burgers | u_t + u u_x = ν u_xx | ν ~ logU[1e-4, 1e-2] |
| reaction_diffusion ("AC") | u_t = D u_xx + r u(1−u), r=1 | D ~ logU[1e-4, 1e-2] |

Evaluation axes (all on the frozen latent, comparable across every method):
coefficient probes, 4-class structure, **consistency under partial
observation**, representation richness (participation ratio, within-field
dispersion, multi-target & IC-mode spectral probes), and **cross-config
transfer** of a frozen probe head.

## Quickstart

```bash
# data (writes data/1d/{system}/{system}_g1.h5, GPU spectral solvers)
python data_gen/gen_g1_all.py

# train the two core methods
python scripts/train_fae.py    --method fae_vicreg --out results/checkpoints/g1/fae_vicreg.pt --gpu 0
python scripts/train_fae_np.py --beta 1e-4 --out results/checkpoints/g1/fae_np_b1e-4.pt --gpu 0

# baselines
python scripts/train_fae.py --method fae_recon --out results/checkpoints/g1/fae_recon.pt
python scripts/train_fae.py --method mlp  --out results/checkpoints/g1/mlp.pt   # also: cnn, mae
python scripts/train_jepa_vit.py       --out results/checkpoints/g1/jepa_vit.pt
python scripts/train_jepa_perceiver.py --out results/checkpoints/g1/jepa_perceiver.pt

# evaluate everything that has a checkpoint (probes, classification, consistency)
python scripts/evaluate.py

# richness / spectral / invariance diagnostics
python scripts/diag_richness.py
python scripts/diag_ic_mode_probe.py
python scripts/diag_cross_config.py

# FAE-NP-specific evaluation (posterior stats, probe-vs-VICReg, ensembles)
python scripts/eval_fae_np.py --np_ckpt fae_np_b1e-4.pt
python scripts/eval_fae_np_ensemble.py --np_ckpt fae_np_b1e-4.pt
```

## Key findings so far (see docs/results/)

- **Detector-vs-encoder**: CNN, MAE and recon-only FAE collapse to effectively
  rank-2 latents that still pass single-coefficient probes; richness
  diagnostics are required to see it. MLPSparseAE is a pure scalar-statistic
  detector (energy R²=1.0, burgers ν R²≈0, IC-mode R²≈0).
- **FAE+VICReg is the best balance**: healthy participation ratio, lowest
  within-field dispersion (true discretization invariance), and the only
  method (with the T2 variant) whose frozen probe head transfers across
  sensor count and jitter with ΔR² ≈ 0. Clustered-layout shift breaks every
  method — open frontier.
- **Advection β is structurally unidentifiable from a single snapshot** for
  every encoder — an honest negative result, not a model failure.
- **FAE-NP status**: trains stably with MSE-recon + free-bits, but the
  posterior sits at the logvar clamp upper bound (σ uninformative) and the
  linear coefficient probe trails FAE+VICReg while the MLP probe is
  comparable — the latent is informative but not linearized. Open thread.

## Conventions (do not break)

1. **Fair comparison**: never compare reconstruction accuracy across methods
   trained with different objectives; probes/consistency/classification on
   frozen latents are the cross-method currency.
2. **Shuffle before split** — `src.data.g1.train_val_split`.
3. **Per-snapshot encoding** (mid-frame), 1024-point native grid.
4. ~**7M parameter parity** for any new method (≤10% deviation).
5. Dense-input methods get **n/a** for consistency metrics — no zero-fill
   adapters in the headline comparison.
